# Output Format Guidelines

Structure all outputs using the following format:

## Summary
One paragraph executive summary of findings, suitable for a non-technical stakeholder.

## Findings
Each finding must include:
- **Severity**: Critical / High / Medium / Low / Informational
- **Description**: What was found and where
- **Evidence**: Supporting data, log entries, or code references
- **Recommendation**: A concrete, actionable remediation step

## Risk Assessment
Overall risk rating (Critical / High / Medium / Low) with a one-sentence rationale.

## Next Steps
Ordered list of recommended actions. Each item should name an owner (e.g., "Security team", "Dev team", "On-call engineer") and a suggested timeframe (immediate, 24h, 1 week).
